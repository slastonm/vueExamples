The program we just created can automatically retrieve info stored in database and display them in a list.

We can also edit, create and delete those information as well.

However, this program has lots of room for improvements.

For example, the codes we use to create and update items are very similar.

They both have two input boxes, one selection list and two radio buttons.

They both save user-submitted data to a Vue data property.

They both send a Vue data property to the backend file.  

They also have the same data filtering mechanism.

In terms of dealing with feedback from backend file, they do exactly the same thing: calling the retrieve all method to update DOM or displaying an error message.

So can we just write the code once and reuse it?

Yes, we can.

A little hint for you: component.

We create one component and use it for creating and editing purposes.

Sending data into a component requires props.

Using prop, we can decide whether a component is for creating new item or updating existent ones.

Using Ajax, component methods can communicate directly with the backend file.

When feedback from backend file has reached the component method, we can tell Vue instance using self-defined event and the emit method.

If everything is fine, we tell Vue instance to call the retrieve all method. If something goes wrong, we tell the Vue instance to display an error message.

New Vue methods and data properties may be required.
